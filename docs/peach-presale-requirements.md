# 桃子预售小程序需求规格

## 1. 项目目标

建设一个用于桃子预售的微信小程序，支持商品预售、库存控制、自提点选择、快递信息、订单管理、打印、优惠码追踪、分享裂变和批量手机号导入等能力。

## 1.1 已确认业务决策

- 支付：未来对接微信支付；MVP 阶段先做模拟支付成功效果，不调用真实微信支付接口。
- 库存：已升级为规格库存；盒装、袋装分别维护库存，商品总库存为各规格库存合计。
- 管理后台：做在微信小程序内部；买家端只展示“管理员登录”入口，不直接展示运营能力；进入运营端前需通过本地管理员手机号校验，正式版替换为后端/openid 权限。
- 手机号导入：用于白名单和折扣人群；用户通过微信授权绑定手机号后，命中手机号全站展示专属折扣价。
- 白名单折扣示例：导入某手机号后，该用户进入小程序看到全站 8 折价格提示。文档只保留脱敏示例，真实手机号不写入代码或文档。

## 2. 用户角色

| 角色 | 主要能力 |
| --- | --- |
| 买家 | 浏览最新上架商品、选择袋装/盒装、自提点或快递、下单、使用优惠码、分享商品 |
| 管理员/运营 | 在小程序内管理后台上传/上架/下架商品、维护库存、自提点、导入手机号、上传快递信息、打印订单、查看优惠码追踪 |
| 自提点工作人员（可选） | 查看本站订单、核销自提订单 |

## 3. 核心业务规则

### 3.1 商品管理

- 商品支持上传/创建。
- 商品支持上架、手动下架、删除/隐藏。
- 商品所有规格库存合计归 0 时自动下架。
- 库存按规格维护：盒装、袋装分别入库、扣减和展示，商品总库存自动汇总。
- 包装类型与配送方式在管理后台必须支持多选，允许一个商品同时支持盒装和袋装、自提和快递。
- 商品至少包含：
  - 商品名称
  - 商品主图/展示图
  - 包装类型：袋装、盒装
  - 售价
  - 折扣价/活动价
  - 库存
  - 预售说明
  - 配送方式：自提、快递或两者都支持
  - 上架时间
  - 状态：草稿、上架、售罄自动下架、手动下架

### 3.2 自提点位设置

- 管理员维护自提点列表。
- 自提点至少包含：
  - 名称
  - 地址
  - 经纬度
  - 联系电话
  - 营业/自提时间
  - 支持包装：袋装、盒装
  - 是否启用
- 买家选择自提时，按距离最近排序。
- 若用户未授权定位，按默认排序或提示手动选择区域。

### 3.3 快递信息上传

- 订单可绑定快递公司、快递单号。
- 支持单条上传和后续批量上传扩展。
- 买家订单详情可查看快递信息。

### 3.4 订单与打印

- 订单流程：待支付/模拟已支付/待自提/待发货/已发货/已完成/已取消/退款中（可选）。
- MVP 阶段点击支付后直接模拟支付成功，并进入后续履约状态；后续替换为真实微信支付回调。
- 订单包含：
  - 买家手机号
  - 商品与规格
  - 数量
  - 包装类型
  - 自提点或快递地址
  - 优惠码
  - 实付金额
  - 备注
- 订单打印需要支持：
  - 自提单
  - 快递面单/发货单
  - 按商品/自提点/日期筛选后批量打印

### 3.5 优惠码追踪

- 支持创建优惠码。
- 优惠码可配置：
  - 折扣金额或折扣比例
  - 使用次数限制
  - 每个手机号使用次数
  - 有效期
  - 适用商品/全场
  - 来源标记，如团长、渠道、员工
- 订单记录优惠码来源，方便追踪转化。

### 3.6 分享逻辑

- 小程序默认分享最新上架商品。
- 若当前页面是商品详情，则优先分享当前商品。
- 若没有上架商品，分享小程序首页。

### 3.7 批量手机号导入

- 管理员可导入手机号列表。
- 用途：白名单、老客户通知、指定折扣人群或营销追踪。
- 需校验手机号格式、去重、记录导入批次。

### 3.8 价格与折扣设计

- 支持全场折扣配置。
- 支持商品级折扣。
- 支持优惠码折扣。
- 支持手机号白名单折扣，例如白名单用户全站 8 折。
- 白名单折扣应在用户注册/登录并绑定手机号后生效。
- 页面价格展示需要区分普通价和白名单专属价，并提示“白名单专享 8 折”等文案。
- 推荐价格计算顺序：
  1. 商品原价 × 商品折扣/活动价
  2. 应用白名单/折扣人群折扣
  3. 应用全场折扣
  4. 应用优惠码
  5. 得到实付金额
- 每一步折扣都需要在订单中留痕，方便对账。

## 4. 推荐页面结构

### 买家端小程序

1. 首页
   - 最新上架商品
   - 默认分享商品入口
   - 活动说明
2. 商品详情页
   - 商品图、规格、袋装/盒装、库存、预售说明

3. 下单页

2. 库存管理
3. 自提点管理

5. 快递信息上传

7. 优惠码管理与追踪

## 6. 图片生成规则

- 商品主图/展示图优先使用 Kie `gpt-image-2-text-to-image` 开发期生成。
- MVP 管理后台支持从相册/相机选择商品主图，保存到本机 mock 商品数据；未选择时使用默认生成图兜底。
- API Key 只允许作为本地环境变量或后端密钥，不允许写入小程序前端代码。
- 生成图片保存到 `miniprogram/assets/images/generated/`。
- 展示清单写入 `miniprogram/utils/generated-images.js`。

## 7. 待确认问题

1. 是否需要微信支付？若要真实收款，需要商户号和后端下单接口。
2. 管理员后台做在微信小程序里，还是做独立网页管理台？
3. 订单打印目标是什么：普通 A4 打印、热敏小票、快递电子面单，还是都需要？
4. 快递信息上传是手动输入、Excel 导入，还是对接快递平台？
5. 批量手机号导入用于白名单、优惠人群，还是短信/群发通知？
6. 自提点是否需要核销码/扫码核销？
7. 白名单折扣默认是否固定 8 折，还是每个导入批次可配置不同折扣？
8. 管理员身份如何判定：指定管理员手机号、微信 openid 白名单，还是开发期先用 mock 开关？

## 8. 简易商城系统补充需求

当前 MVP 已覆盖预售下单主链路，但作为简易商城还需要补齐以下能力：

### 8.1 底部导航

- 首页：活动与最新上架商品。
- 商品：全部商品列表、状态筛选、价格展示。
- 订单：用户订单列表、订单状态、快递/自提摘要。
- 我的：微信登录与手机号绑定、白名单折扣状态、地址入口、客服说明、管理员身份校验入口。

### 8.2 个人中心

- 显示当前绑定手机号（脱敏）。
- 显示白名单折扣状态，例如“白名单专享 8 折”。
- 我的订单入口。
- 收货地址管理入口，支持本地常用地址、默认地址，下单快递可套用。
- 联系客服/售后说明。
- “我的”页面可展示“管理员登录”入口，但必须先校验管理员身份；普通用户不能直接进入运营端。

### 8.3 订单中心

- 展示订单列表。
- 展示订单详情：商品规格、金额明细、优惠留痕、配送/地址、订单基础信息。
- 展示订单状态：模拟已支付、待自提、待发货、已发货、已完成、已取消。
- 展示配送方式：自提点或快递地址摘要。
- 后续应支持取消订单、申请售后、确认收货/核销。

### 8.4 商品列表与搜索筛选

- 全部商品列表。
- 状态筛选：上架中、售罄、已下架。
- 包装筛选：袋装、盒装。
- 后续可加关键词搜索、价格排序、销量排序。

### 8.5 购物车与复购（后续）

- 当前 MVP 是“立即下单”模式。
- 后续商城化建议增加购物车、收藏、再次购买。

### 8.6 地址管理

- 当前 MVP 支持本地常用地址保存、默认地址、下单快递套用。
- 后续建议接入微信地址选择 API，并把地址同步到后端账户。

### 8.6.1 SKU/规格价格与库存

- 商品总库存由各规格库存合计得出。
- 规格可独立配置名称、包装、重量、原价、预售价、库存、支持配送方式。
- 下单时只扣减当前选择规格的库存；所有规格库存都为 0 时商品自动售罄下架。
- 价格计算基于当前选中的规格，再叠加白名单折扣、全场折扣、优惠码。

### 8.7 售后与客服（后续）

- 退款/补发规则。
- 联系客服按钮。
- 自提过期处理规则。

### 8.8 用户端/运营端隔离与管理员权限（后续必须）

- 买家端底部导航只包含：首页、商品、订单、我的。
- 买家端个人中心可展示“管理员登录”入口，但入口只做身份校验，不展示商品/订单/库存管理能力。
- MVP 阶段使用本机管理员手机号 storage 校验：本机无管理员时，当前已登录手机号首次进入运营端会自动设为本机管理员；开发期可追加本机管理员，正式版必须关闭该能力并改为服务端权限。
- 运营端保留在小程序内部独立页面路径，也可通过管理员登录入口打开。
- 正式版必须通过管理员手机号/openid 白名单或服务端 RBAC 控制，否则普通用户可能进入管理页。
