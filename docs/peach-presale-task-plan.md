## 建议 MVP 顺序

1. 买家端首页 + 商品详情 + Kie 商品图展示。
2. 商品 mock 数据 + 库存归零自动下架逻辑。
3. 自提点选择 + 距离排序。
4. 下单表单 + 优惠码价格计算。
5. 管理端能力：商品、订单、快递、打印、手机号导入。

## 当前可立即执行的第一批开发任务

1. 建立 `miniprogram/data/` mock 数据：商品、自提点、优惠码、折扣配置、白名单手机号批次。
2. 建立 `miniprogram/utils/pricing.js`：统一价格计算，包含白名单 8 折、全场折扣、优惠码。
3. 建立 `miniprogram/utils/inventory.js`：总库存状态与自动下架。
4. 建立 `miniprogram/utils/distance.js`：自提点距离排序。
5. 扩展首页为桃子预售首页，引用最新上架商品和 Kie 配图。
6. 增加微信登录与手机号授权绑定入口；本地 MVP 保留开发期手机号绑定兜底，命中白名单后全站切换为折扣价展示。
7. 增加模拟支付成功流程，预留未来微信支付替换点。
