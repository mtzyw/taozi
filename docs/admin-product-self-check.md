# 管理后台商品模块自查

## 本轮发现的问题

1. 新增商品后没有删除能力，运营端无法纠错。
2. 包装类型和配送方式设计成单选，不符合一个商品同时支持盒装/袋装、自提/快递的业务场景。
3. 库存操作最初只有商品总库存，不支持盒装/袋装分别入库。
4. 商品管理卡片信息不足，缺少包装、配送、来源等运营判断信息。
5. “设为库存”如果空输入误点，可能被当成 0，容易导致商品误下架。
6. 新增商品缺少主图上传入口，运营无法给预售商品配置真实图片。

## 已更新

1. 管理后台新增商品表单改为包装多选：盒装、袋装可同时选择。
2. 配送方式改为多选：自提、快递可同时选择。
3. 商品与库存列表增加商品来源、包装、配送、原价、预售价、总库存、状态。
4. 库存升级为 SKU 级管理：
   - 盒装、袋装分别展示库存。
   - 设为该规格库存：只修改当前规格库存。
   - 补货该规格：只给当前规格累加入库。
   - 商品总库存由各规格库存自动汇总。
5. 增加删除商品能力：
   - 后台创建商品会从本地自定义商品中移除。
   - 系统示例商品会在本机 mock 数据中隐藏。
6. 删除前增加确认弹窗，避免误删。
7. “设为该规格库存”增加空值/负数校验，成功后清空输入框，降低误操作风险。
8. 新增商品表单增加商品主图上传/选择/预览/移除；不上传时仍使用默认配图兜底。

## 仍需后续完善

1. 图片目前保存在本机 mock/本地文件路径，正式版应上传到云存储/CDN 并保存远程 URL。
2. 暂未做商品编辑页；当前可通过删除后重建、库存调整、上下架完成基础运营。
3. 正式版应由后端做软删除、操作日志、管理员权限和图片资产管理。
